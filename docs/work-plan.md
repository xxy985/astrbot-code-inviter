# Work Plan

## 目标

把 `astrbot_plugin_code_inviter` 从 PRD 推进到可安装、可验证的 AstrBot 插件。

## 阶段拆分

1. 仓库和插件骨架
   - 建立 AstrBot 插件元数据和入口文件。
   - 确认配置 schema、数据目录和测试结构。

2. 数据层
   - 初始化 SQLite 表结构。
   - 实现码池、码、领取记录、待加好友流程和禁领用户的数据访问。
   - 用事务保证同一个码不会被并发重复发放。

3. 配置和匹配
   - 解析全局配置和码池配置。
   - 实现整句精确匹配。
   - 处理全局 allowed groups 与码池级覆盖。

4. 群触发和好友验证
   - 群触发后生成一次性 token。
   - 记录待处理好友流程。
   - 处理好友申请事件并自动通过或拒绝。

5. 私聊领取
   - 根据私聊触发词定位码池。
   - 校验来源、禁领、库存、冷却和领取规则。
   - 发放一个未使用码并记录领取事件。

6. 管理员命令
   - 实现文本导入、CSV 导入、库存查询、统计查询、用户领取查询和 CSV 导出。
   - 限制命令只能由 `admin_users` 使用。

7. 验证和收口
   - 增加单元测试和必要的集成测试。
   - 补充安装说明、配置示例和常见问题。
   - 进行手动 AstrBot + NapCat 环境验证。

## 本轮不做

- 不实现完整插件业务逻辑。
- 不连接真实 AstrBot / NapCat 实例，也不扩展到其他平台或接入层。
- 不上传本地 Trellis / Codex / agent 工作流文件。

## 验证标准

- 每个阶段都有可运行的测试或可复核的手动验证步骤。
- 涉及发码的数据变更必须可追溯到数据库记录。
- CSV 导入导出默认使用 `utf-8-sig`。
- 非管理员不能执行管理命令。
