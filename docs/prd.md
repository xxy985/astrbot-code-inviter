# PRD: AstrBot 多码池加好友发码插件

## 1. 产品概述

### 1.1 产品名称

暂定：`astrbot_plugin_code_inviter`

### 1.2 产品目标

为 AstrBot 插件场景开发一个多码池发码插件。AstrBot 通过 NapCat 接入机器人后，插件通过指定 QQ 群触发词引导用户添加机器人好友，机器人校验群来源和一次性验证 token 后自动通过好友申请，用户再通过私聊触发词每次领取一个邀请码、兑换码或其他类型码。

插件需要同时满足：

- 普通用户流程简单，不暴露复杂规则。
- 管理员可以批量导入码、查询库存、导出 CSV、查看统计。
- 每个码池可以独立配置触发词、领取规则、统计规则和消息文案。
- 通过群来源校验降低陌生人直接私聊薅码的风险。

### 1.3 目标用户

| 用户类型 | 典型角色 | 核心诉求 |
| --- | --- | --- |
| 普通领取用户 | QQ 群成员 | 按提示添加机器人好友，并私聊领取指定类型的码 |
| 插件管理员 | 群主、运营、发码负责人 | 管理码池、导入码、查询库存、导出领取记录、处理异常用户 |
| 系统维护者 | 机器人部署者 | 在 AstrBot 后台配置规则，保证插件稳定运行 |

## 2. 需求追溯链

| 目标 | 用户 | 场景 | 问题 | 证据 / 假设 | 需求 | 验收标准 | 验证信号 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 自动化发放多种码 | 普通领取用户 | 用户在群内看到活动后想领取邀请码或兑换码 | 人工私聊发码耗时，容易漏发、重复发 | 假设：当前发码需要管理员手动处理 | 用户能通过群触发 + 私聊领取指定码池的一个码 | 用户按流程能成功获得正确码池的一个未使用码 | 人工发码次数减少，成功领取率提升 |
| 限制领取来源 | 管理员 | 只希望指定群成员领取 | 陌生人可能直接私聊机器人领码 | 假设：QQ 私聊入口可能被转发或滥用 | 私聊领取前必须有指定群触发记录 | 未在允许群触发的用户不能领取 | 非目标来源领取被拦截 |
| 支持多码池 | 管理员 | 同时发放邀请码和兑换码 | 单码池不能满足多活动、多权益发放 | 已确认至少需要两种码 | 每个码池独立库存、触发词和领取规则 | 两个以上码池可并行使用且互不串码 | 各码池库存、领取记录可独立查询 |
| 控制领取规则 | 管理员 | 某些码只能领一次，某些可多次领取 | 全局一次领取规则不够灵活 | 已确认每个码池规则不同 | 每个码池可设置一次、限次、周期限次或不限次 | 插件按码池规则准确允许或拒绝领取 | 重复领取、周期领取符合配置 |
| 支持统计和导出 | 管理员 | 多次领取码池需要外部统计 | 没有记录无法分析消耗、异常用户和批次效果 | 已确认需要 CSV 导出 | 记录每次领取事件，并支持导出 CSV | 管理员能导出指定码池、日期范围的 CSV | CSV 可用于表格软件统计 |

## 3. 范围

### 3.1 v1 必须包含

- 多码池管理，至少支持两个码池。
- 群聊精确触发词引导用户添加好友。
- 为每次群触发生成一次性好友验证 token。
- 自动处理好友申请：仅通过有有效群触发记录和匹配 token 的用户。
- 私聊精确触发词领取码，每次触发只发一个码。
- 每个码池独立领取规则。
- 文本列表批量导入码。
- CSV 批量导入码。
- CSV 导入支持本地服务器文件路径；如果 QQ 文件消息解析能力可用，也支持从 QQ 文件导入。
- 本地生成 CSV 领取记录导出文件。
- 管理员库存查询、统计查询、用户领取查询。
- SQLite 持久化码池、码、领取记录、待加好友流程和黑名单。
- AstrBot 后台配置基础规则、码池规则和消息模板。

### 3.2 v1 不包含

- 独立 Web 管理后台。
- 图形化上传 CSV。
- 多机器人共享库存。
- 支付系统或订单校验。
- 机器人主动添加用户好友。
- 复杂活动系统、抽奖系统、积分系统。
- NapCat 发送导出文件给管理员作为必选能力。
- 管理员补发功能。
- 每个码池的每日总发放上限。

## 4. 核心流程

### 4.1 群内触发加好友流程

1. 用户在允许群发送码池的群触发词，例如 `领邀请码`。
2. 插件使用整句精确匹配识别触发词。
3. 插件检查该群是否允许领取该码池。
4. 插件生成一次性验证 token，并记录待处理流程：
   - `user_id`
   - `group_id`
   - `pool_id`
   - `verify_token`
   - `expires_at`
5. 机器人在群内回复加好友说明，例如：`请添加机器人好友，验证信息填写：领码-482913，然后私聊发送：领取邀请码`。

### 4.2 自动通过好友申请

1. 用户添加机器人好友，并填写验证信息。
2. NapCat 上报好友请求事件给 AstrBot。
3. 插件校验：
   - 申请人是否有未过期待处理流程。
   - 验证信息是否匹配该用户的 token。
   - 触发来源群是否在允许范围内。
   - 目标码池是否启用。
4. 校验通过后调用 AstrBot 通过 NapCat 暴露的能力同意好友申请。
5. 校验失败时不自动通过，并记录日志。

### 4.3 私聊领取码

1. 用户私聊机器人发送码池私聊触发词，例如 `领取邀请码`。
2. 插件精确匹配触发词并定位码池。
3. 插件检查：
   - 用户是否有有效群来源记录。
   - 用户是否被禁领。
   - 码池是否启用。
   - 领取规则是否允许本次领取。
   - 是否有可用库存。
4. 插件从对应码池取出一个未使用码。
5. 插件发送私聊消息：`你的邀请码是：{code}`。
6. 插件记录本次领取事件。

## 5. 功能需求

### 5.1 码池管理

每个码池应支持：

- 唯一 ID，例如 `invite`、`redeem`。
- 显示名称，例如 `邀请码`、`兑换码`。
- 启用 / 禁用。
- 群触发词列表。
- 私聊触发词列表。
- 允许群配置。
- 领取规则。
- 消息模板。码池未配置独立文案时使用全局消息模板。
- 统计开关。

触发词默认使用整句精确匹配。v1 可预留正则配置，但默认关闭。

### 5.2 领取规则

每个码池独立配置领取规则：

| 模式 | 含义 |
| --- | --- |
| `once_per_user` | 每个用户在该码池只能领取一次 |
| `limited_per_user` | 每个用户在该码池总计最多领取 N 次 |
| `limited_per_period` | 每个用户在每天、每周或每月最多领取 N 次 |
| `unlimited` | 不限制总次数，但仍可设置冷却时间 |

通用限制：

- 每次主动触发只领取一个码。
- 每次领取消耗一个未使用码。
- 不重复发放同一个码。
- 允许多次领取的码池必须记录每一次领取事件。

### 5.3 批量导入

支持两种导入方式：

1. 管理员文本列表导入：

```text
/导入码 邀请码
CODE001
CODE002
CODE003
```

2. CSV 导入：

```csv
code,batch,remark
CODE001,2026-05-A,第一批
CODE002,2026-05-A,第一批
```

导入要求：

- 同一码池内 code 不允许重复。
- 导入时统计成功、重复、失败数量。
- 支持 `utf-8-sig` 编码，避免 Excel 打开乱码。
- 每条码可记录批次和备注。
- CSV 可通过本地服务器文件路径导入；如果适配器能解析 QQ 文件消息，也允许从 QQ 文件导入。

### 5.4 CSV 导出

v1 采用本地生成 CSV 文件路径的方式。

管理员命令示例：

```text
/导出领取记录 邀请码
/导出领取记录 邀请码 今日
/导出领取记录 邀请码 2026-05-01 2026-05-21
```

导出字段：

```csv
record_id,pool_id,pool_name,user_id,user_nickname,source_group_id,source_group_name,code,batch,claimed_at,claim_index,status
```

导出要求：

- 支持按码池导出。
- 支持按日期范围导出。
- 导出文件保存到插件数据目录下的 `exports`。
- 机器人回复管理员本地文件路径和记录数量。

### 5.5 管理员命令

建议 v1 支持：

```text
/发码库存
/发码库存 邀请码

/发码统计
/发码统计 兑换码

/查领取 123456789
/查领取 123456789 邀请码

/导入码 邀请码
/导入csv 邀请码 codes.csv

/导出领取记录 邀请码
/重置领取 邀请码 123456789

/禁领 123456789
/解禁 123456789
```

管理员命令只允许配置中的 `admin_users` 使用。

## 6. 后台配置

AstrBot 后台配置主要承载规则和文案，不承载大量码数据。

建议配置结构：

```yaml
enabled: true

admin_users:
  - 123456789

global_allowed_groups:
  - 987654321

matching:
  trigger_match_mode: "exact"
  enable_regex: false

friend_gate:
  auto_approve: true
  require_group_trigger_before_approve: true
  require_verify_token: true
  token_ttl_minutes: 30
  token_template: "领码-{token}"
  reject_without_pending_flow: true

claim_gate:
  require_group_source: true
  group_source_ttl_hours: 24
  allow_existing_friends_without_group_source: false

code_pools:
  invite:
    enabled: true
    display_name: "邀请码"
    allowed_groups: []
    group_triggers:
      - "领邀请码"
      - "我要邀请码"
    private_triggers:
      - "领取邀请码"
    claim_policy:
      mode: "once_per_user"
      per_user_limit: 1
      period: "none"
      period_limit: 0
      cooldown_seconds: 30
    statistics:
      enabled: true
    messages: {}

  redeem:
    enabled: true
    display_name: "兑换码"
    allowed_groups: []
    group_triggers:
      - "领兑换码"
      - "我要兑换码"
    private_triggers:
      - "领取兑换码"
    claim_policy:
      mode: "limited_per_period"
      per_user_limit: 0
      period: "day"
      period_limit: 1
      cooldown_seconds: 60
    statistics:
      enabled: true
    messages: {}

csv:
  encoding: "utf-8-sig"
  export_dir: "exports"
  include_code_value: true

messages:
  group_guide: "请添加机器人好友，验证信息填写：{verify_text}，然后私聊发送：{private_trigger}"
  code_message: "你的{pool_name}是：{code}"
  already_claimed: "你已经领取过{pool_name}了。"
  limit_reached: "你当前已达到{pool_name}领取上限。"
  out_of_stock: "{pool_name}当前库存不足。"
  not_allowed_group: "当前群暂不支持领取{pool_name}。"
  not_friend_flow: "请先在指定群触发领取流程。"
  cooldown: "操作太频繁，请稍后再试。"
  pool_disabled: "{pool_name}暂未开放领取。"

logging:
  log_claims: true
  log_imports: true
  log_friend_requests: true
```

码池级 `allowed_groups` 为空时使用 `global_allowed_groups`；不为空时覆盖全局配置。

码池级 `messages` 为空时使用全局 `messages`；如果配置了同名消息键，则只覆盖该码池对应文案。

## 8. 验收标准

### 8.1 用户流程验收

- 在允许群发送精确触发词后，机器人返回正确加好友说明。
- 未在允许群触发的用户无法通过私聊领取。
- 用户使用正确 token 添加好友时，机器人自动通过申请。
- 用户使用错误验证信息或过期 token 时，机器人不自动通过。
- 私聊发送码池触发词后，用户每次只收到一个未使用码。
- 领取规则达到上限时，机器人返回明确提示。

### 8.2 管理员功能验收

- 管理员可以向指定码池批量导入文本码。
- 管理员可以从 CSV 导入码，并看到成功、重复、失败统计。
- 管理员可以查询全部码池和单个码池库存。
- 管理员可以查询用户领取记录。
- 管理员可以导出指定码池的 CSV 领取记录。
- CSV 领取记录包含用户昵称和来源群名快照，便于外部统计。
- 非管理员无法执行管理命令。

### 8.3 数据一致性验收

- 同一码池内不会重复导入相同码。
- 同一个码不会被两个用户同时领取。
- 每次成功领取都产生一条 `claim_records`。
- 多次领取码池能正确统计用户累计领取次数。
- CSV 导出记录与数据库领取记录一致。

## 9. 验证信号

上线后观察：

- 成功领取次数。
- 领取失败原因分布。
- 各码池库存消耗速度。
- 重复领取拦截次数。
- 无群来源私聊拦截次数。
- 好友申请自动通过成功率。
- CSV 导出使用次数。

如果人工发码明显减少、重复发码率降低、管理员能通过 CSV 完成统计，则 v1 达到目标。

## 10. 风险与处理

| 风险 | 影响 | 处理 |
| --- | --- | --- |
| NapCat 好友申请事件不提供可靠群来源 | 无法判断是否从群加好友 | 使用插件自己的群触发记录和一次性 token 判断 |
| 用户复制他人 token | 可能冒领流程 | token 绑定 `user_id`，验证时同时校验申请人 |
| 并发领取导致重复发码 | 同一个码可能发给多人 | SQLite 事务锁定未使用码并立即标记 |
| 触发词误判 | 群聊中普通讨论触发机器人 | 默认整句精确匹配，不做模糊包含 |
| CSV 编码乱码 | 管理员统计困难 | 默认 `utf-8-sig` 导出 |
| 后台配置过复杂 | 管理员不会用 | 后台只配置规则，码池数据通过命令导入 |

## 11. 已确认决策

- 消息文案支持全局模板和码池级覆盖；码池未配置时使用全局模板。
- CSV 导入同时支持本地服务器文件路径和 QQ 文件消息解析；具体实现以 AstrBot / NapCat 文件事件能力为准。
- v1 不需要每个码池的每日总发放上限。
- 领取记录需要保存用户昵称、来源群名等快照信息，便于 CSV 人工识别。
- v1 不涉及管理员补发功能。

## 12. v1 开发拆分

1. 插件骨架和 AstrBot 配置 schema。
2. SQLite 数据层和迁移初始化。
3. 多码池配置加载和触发词精确匹配。
4. 群触发记录和一次性 token 生成。
5. 好友申请事件处理和自动通过。
6. 私聊领取流程和领取规则判断。
7. 文本列表导入和 CSV 导入。
8. 库存、统计、查询、导出 CSV 管理命令。
9. 日志、异常提示和基础测试。
